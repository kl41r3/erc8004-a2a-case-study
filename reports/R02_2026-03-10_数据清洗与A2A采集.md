## 二、Google A2A 数据采集（`scripts/scrape_a2a.py`）

### 背景

Google A2A 是本研究的比较案例 B（公司科层制治理）。ERC-8004 论文明确指出其扩展了 Google A2A 协议，且 A2A 的联合作者 Jordan Ellis（Google）参与了 ERC-8004 的社区讨论，构成天然对照实验。

### 来源

仓库: https://github.com/google/A2A

采集内容:
- Commits（`GET /repos/google/A2A/commits`）
- Issues + issue comments（`GET /repos/google/A2A/issues?state=all` + `/issues/comments`）
- PRs + PR review comments（`GET /repos/google/A2A/pulls?state=all` + `/pulls/comments`）

### 技术问题与解决

**双问号 bug**: `fetch_paginated(base_url)` 在 `base_url` 已含 `?state=all` 时，继续拼接 `?per_page=100&page=N` 形成无效 URL（`pulls?state=all?per_page=100`）。修复：检测 `?` 是否已存在，用 `&` 替代第二个 `?`。

**数据覆盖事故**: 修复 bug 后重新运行时 GitHub 匿名 API 速率已耗尽（60 req/hr），重跑只得 100 commits、0 PRs，并覆盖了原文件。解决方案：使用 GitHub Personal Access Token（5000 req/hr），完整重采。

**超时导致缺页**: 初次采集 `--max-time 30` 太短，issue comments 在第 9 页（900 条）超时，PR review comments 在第 10 页（1000 条）超时。实际数据有 26 页（2600 条 issue comments）和 12 页（1200 条 PR review comments）。

→ 补漏脚本：`scripts/patch_a2a_missing_pages.py`，从已知缺失页开始，用 `--max-time 90` 重跑，按 `comment_id` 去重合并。

### 最终数据（PAT + 补漏后）

| 类型 | 数量 | 备注 |
|------|------|------|
| Commits | 522 | 2025-03-25 → 2026-03-06；commit 作者 131 人 |
| Issues | 506 | google/A2A Issues 功能实际可用（PAT 解锁） |
| Issue comments | 2,598 | 26 页；最终合并入 a2a_issues.json |
| PRs | 755 | 500 个已合并（merge rate 66.2%）|
| PR review comments | 1,200 | 12 页 |
| **总讨论记录** | **4,923** | |
| 不重复讨论贡献者 | 596 | |

**关键发现**:
- A2A 首个 commit 为 2025-03-25，公开宣布为 2025-04-09，**内部开发提前 15 天**
- Issues 功能开放（非关闭），506 个 issues，说明存在真实社区参与
- PR merge rate 66.2%，755 个 PR 中有 500 个被合并，外部贡献通道实际存在

## 三、LLM 标注脚本重构（`annotate_llm.py` v2）

### 改动

原版硬编码 Anthropic。新版支持三路 backend，一键切换：

| Backend | 接口 | 模型 |
|---------|------|------|
| `minimax`（默认）| `api.minimaxi.com/v1` | MiniMax-M2.5 |
| `openai` | 任意 OpenAI-compat base_url | 可配置 |
| `anthropic` | Anthropic SDK | 可配置 |

通过 `--backend` 参数或 `ANNOTATION_BACKEND` 环境变量切换。支持中断续跑。

## 四、指标计算重构（`compute_metrics.py` v2）

### 改动

1. **分离结构指标和标注指标**：结构指标（参与人数、时间线、讨论量）不依赖 LLM，现在即可计算；标注指标（机构归属、Openness index）在标注完成后自动填充。

## 六、Limitations（本报告新增）

**L1: ERC-8004 GitHub 讨论极稀疏**
核心 lifecycle PR 仅 11 条评论，2 个关键 PR（#1170 初始提案、#1244 Move to Review）有 0 条。治理讨论实际发生在论坛，PR 是行政步骤。从研究设计角度，这不是缺陷，而是 DAO 治理形式的真实反映。

**L2: A2A 内部讨论不可见**
A2A 的真实治理决策（架构选型、路线图）发生在 Google 内部系统（Buganizer、内部文档），不可抓取。外部可见的仅为 commit 和 PR review。这是企业治理对比研究的系统性局限，需在论文 Limitations 节明确说明。

**L3: LLM 标注待运行**
MiniMax-M2.5 已验证可用（见 R03），但本报告截止时标注未完成。机构归属、Openness index 均为 TBD。
