## 一、工作内容

### 1.1 项目基础设施

- 使用 `uv` 在 `workspace/` 创建虚拟环境（Python 3.14.3），避免污染系统 Python
- 安装依赖：`requests`、`beautifulsoup4`、`anthropic`、`pandas`、`openai`、`python-dotenv`、`python-dateutil`
- `.env` 列入 `.gitignore`，API key 不入库

### 1.2 ERC-8004 论坛数据采集（`scripts/scrape_erc8004.py`）

**来源**: https://ethereum-magicians.org/t/erc-8004-trustless-agents/25098

**方法**: Discourse JSON API（`/t/{id}.json` 获取元数据和第一批帖子，`/t/{id}/posts.json?post_ids[]=...` 分批获取其余帖子）

**技术问题与解决**:
- Python 3.14 的 `requests` 库与 ethereum-magicians.org 存在 SSL EOF 错误（`SSLEOFError: UNEXPECTED_EOF_WHILE_READING`），但 `curl` 正常工作。改用 `subprocess` 调用系统 `curl` 解决。
- Discourse 分页 URL 必须用纯数字 topic ID（`/t/25098/posts.json`），加 slug 会返回 404。

**结果**: 113 条帖子，60 个不重复作者，时间跨度 2025-08-14 → 2026-03-10（完整覆盖）

### 1.3 ERC-8004 GitHub 数据采集（`scripts/scrape_erc8004.py`）

**来源**: https://github.com/ethereum/ERCs

**方法**: GitHub Search API（`q=ERC-8004 repo:ethereum/ERCs`）找到相关 PR，再拉取每个 PR 的 issue comments、review comments、reviews。

**结果（原始）**: 149 条，来自 29 个 PR。

**问题**: 搜索结果包含大量生态系 PR——它们只是在规范文档中引用 ERC-8004，并非 ERC-8004 自身的治理讨论（详见 R02 报告中的清洗工作）。

### 1.4 LLM 标注脚本（`scripts/annotate_llm.py` v1）

初版硬编码 Anthropic API，使用 `claude-haiku-4-5`，每条记录提取：
- `stakeholder_institution`（Google / Coinbase / MetaMask / Ethereum Foundation / Independent / Unknown）
- `argument_type`（Technical / Economic / Governance-Principle / Process / Off-topic）
- `stance`（Support / Oppose / Modify / Neutral / Off-topic）
- `consensus_signal`（Adopted / Rejected / Pending / N/A）
- `key_point`（≤20 词摘要）

支持中断续跑（resume），按 `comment_id` / `post_id` + `date` 去重。

### 1.5 指标计算脚本（`scripts/compute_metrics.py` v1）

计算以下治理指标（ERC-8004）：
- 参与广度：不重复贡献者数、不重复机构数
- 讨论深度：总帖数、平均回复率、跨机构回复数
- 决策速度：从提案到 mainnet 部署天数（169 天）
- 开放性指数：非发起方机构贡献者比例
