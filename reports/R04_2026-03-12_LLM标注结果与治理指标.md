### 错误分析

全部 382 条错误类型均为 `json_parse: Expecting value: line 1 column 1`，即 API 返回空字符串内容。

**原因**：MiniMax-M2.5 的内容安全过滤器（`output_sensitive`）对部分 A2A PR review comments 触发了过滤，导致 `choices[0].message.content` 为空。典型触发内容为自动化 bot 评论（`gemini-code-assist[bot]`、CI 机器人等），含 markdown 告警块（`> [!NOTE]`）或代码引用片段。ERC-8004 的 124 条论坛/GitHub 评论全部为人工撰写，无一触发过滤。

**影响评估**：382 条 / 4,665 条 A2A 记录（8.2%）。这批记录以 bot 评论和 CI 通知为主，对治理分析的核心结论（机构参与、论点类型、立场分布）影响极小——bot 评论本身不代表任何治理主体的立场。

## 三、理论解读初稿

### Williamson Puzzle（再述）

ERC-8004 具有高资产专用性（AI agent 协议，需多方共同实现才有价值）和高不确定性（新型 AI 治理领域，2025 年尚无先例）。交易成本经济学（Williamson 1985）预测：这类情境应由科层制组织主导，而非市场或松散协作。

**实际发生的**：竞争企业（Google、Coinbase、MetaMask）与独立开发者在没有合同约束、没有股权结构的情况下，在 169 天内通过 DAO 流程达成了技术共识。

### 两案例对比的理论含义

| 维度 | ERC-8004 | Google A2A | 理论含义 |
|------|----------|------------|---------|
| 参与广度 | 62 人 / 多机构 | 596 人 / Google 主导（23%）| DAO 参与更分散，但 A2A 绝对人数更多 |
| 开放性 | 1.000 | 0.956 | 两者均高度开放，差异不显著 |
| 决策速度 | 169 天 | 持续迭代 | DAO 有明确里程碑（mainnet），企业无终态 |
| 论点构成 | 技术 76.6%，治理 4.8% | 待分析 | ERC-8004 以技术讨论为核心，而非权力博弈 |
| 修改立场 | 30.6% Modify | 待分析 | 高修改率反映真实审议过程，非橡皮图章 |

### 三个 DAO 治理机制（草稿）

1. **区块链强制承诺**：智能合约替代合同保障，降低事后背叛风险
2. **链上声誉外部化**：公开的参与记录消除搭便车激励
3. **无需许可的参与**：任何人可提案、评论、反对，异质利益相关者协调成本显著低于层级授权流程

## 五、Limitations（本报告新增）

**L1：382 条标注失败（A2A bot 评论）**
内容过滤触发，均为自动化 bot 评论，非治理主体。对核心指标影响可忽略，但应在论文 Methods 节声明标注成功率 92%。

**L2：机构归属完全依赖 LLM 推断**
GitHub 用户名通常不直接暴露所在机构。LLM 通过 bio、邮箱域名、帖子内容等文本线索推断。建议对 ERC-8004 全部 124 条和 A2A 随机抽取 50 条进行人工复核，计算 Cohen's Kappa。

**L3：ERC-8004 Openness Index = 1.0 的解读谨慎性**
该值反映的是 LLM 未识别出 Ethereum Foundation 成员参与，而非 EF 确实未参与。实际上 `davidecrapis.eth`（Olas 协议）是发起人，EF 的 ERC 编辑（如 `@lightclient`）参与了流程审核，但在标注中被归为 Independent。这是机构归属推断的固有局限。

**L4：A2A 讨论性质与 ERC-8004 存在根本差异**
ERC-8004 的 124 条是公开治理辩论（论坛帖子）；A2A 的 4,665 条以 PR review comments（代码审查）为主。两者的"讨论"含义不同，直接数量对比不严谨，应在论文中分别处理。
